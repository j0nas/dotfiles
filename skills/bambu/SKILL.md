## 3MF format reference

A 3MF file is a ZIP archive. Bambu Studio expects this structure:

```
[Content_Types].xml
_rels/.rels
3D/3dmodel.model                    # assembly: top-level objects + build plate items
3D/_rels/3dmodel.model.rels         # references to object model files
3D/Objects/object_N.model           # per-object mesh data (vertices + triangles)
Metadata/model_settings.config      # object/part names + extruder assignments
Metadata/project_settings.config    # full slicer/machine/filament settings
Metadata/slice_info.config          # client version header
Metadata/cut_information.xml        # cut data per object (stub OK)
Metadata/filament_sequence.json     # {"plate_1":{"sequence":[]}}
```

### XML namespace convention

3MF model files MUST use `p:` as the namespace prefix:

```xml
<model unit="millimeter" xml:lang="en-US"
  xmlns="http://schemas.microsoft.com/3dmanufacturing/core/2015/02"
  xmlns:BambuStudio="http://schemas.bambulab.com/package/2021"
  xmlns:p="http://schemas.microsoft.com/3dmanufacturing/production/2015/06"
  requiredextensions="p">
```

Components use `p:UUID`, `p:path`. Build items use `p:UUID`.

### Multi-color objects

Each top-level object in `3dmodel.model` has `<components>` referencing mesh parts in an object file. Each part gets its own extruder assignment in `model_settings.config`:

```xml
<!-- In model_settings.config -->
<object id="3">
  <metadata key="name" value="My Object"/>
  <metadata key="extruder" value="2"/>
  <part id="1" subtype="normal_part">
    <metadata key="name" value="Body"/>
    <metadata key="extruder" value="2"/>
  </part>
  <part id="2" subtype="normal_part">
    <metadata key="name" value="Accent"/>
    <metadata key="extruder" value="1"/>
  </part>
</object>
```

### Avoiding the "old version" warning

Without `Metadata/slice_info.config` and `Metadata/project_settings.config`, Bambu Studio shows a "generated by an old version" warning. Always include both. A saved H2C settings template exists at `C:/Users/jonas/Desktop/bambu/Filament Swatch/h2c_project_settings.config`.

The `slice_info.config` is minimal:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<config>
  <header>
    <header_item key="X-BBL-Client-Type" value="slicer"/>
    <header_item key="X-BBL-Client-Version" value="02.05.00.66"/>
  </header>
</config>
```

### STL to 3MF mesh conversion

To embed STL mesh data in a 3MF object file, parse the STL (ASCII or binary), deduplicate vertices, and write:

```xml
<object id="1" p:UUID="..." type="model">
  <mesh>
    <vertices>
      <vertex x="0.0" y="3.0" z="0.0"/>
      ...
    </vertices>
    <triangles>
      <triangle v1="0" v2="1" v3="2"/>
      ...
    </triangles>
  </mesh>
</object>
```

## Bambu Studio CLI reference

```
bambu-studio.exe [OPTIONS] [file.3mf/file.stl ...]

Arrangement & orientation:
  --arrange 0|1           Auto-arrange objects on plate
  --orient                Auto-orient models
  --scale factor          Scale model

Slicing & export:
  --slice 0|i             Slice plates (0=all, i=plate i)
  --export-3mf file.3mf   Export project as 3MF
  --export-stl            Export as single STL
  --export-stls           Export as multiple STLs

Settings:
  --load-settings "machine.json;process.json"
  --load-filaments "fil1.json;fil2.json"
  --load-filament-ids "1,2,3,1"

Info:
  --info                  Output model info
  --debug 0-5             Logging verbosity
```

### CLI limitations

- Cannot create geometry or add text (GUI only)
- `--uptodate` flag segfaults on programmatically generated 3MFs
- Re-exporting programmatically generated 3MFs via CLI also segfaults -- only the GUI can re-save
- Works fine for slicing and exporting known-good 3MFs

## OpenSCAD notes

- Installed at `C:/Program Files/OpenSCAD/openscad.exe` (version 2021.01)
- CLI rendering: `openscad -o output.stl input.scad`
- Outputs **ASCII STL** by default -- any STL parser must handle both ASCII and binary
- Has excellent built-in `text()` function for 3D text generation (uses system fonts)
- Braces in OpenSCAD code (`{`, `}`) conflict with Python's `.format()` -- use f-strings with doubled braces or string concatenation instead

## Python + 3MF gotchas

### Do NOT use xml.etree.ElementTree for 3MF

Python's ElementTree mangles namespace prefixes (`ns0:UUID` instead of `p:UUID`), producing files Bambu Studio rejects or loads incorrectly. Use string-based XML generation instead.

### OpenSCAD ASCII STL detection

Check for `solid` keyword at start AND `facet` within first 1000 bytes. Some binary STLs happen to start with `solid` in the header, so checking both avoids misdetection.

### Filament swatch generator

A working script at `C:/Users/jonas/Desktop/bambu/Filament Swatch/generate_swatches.py` generates multi-color filament sample swatches.

```bash
cd "C:/Users/jonas/Desktop/bambu/Filament Swatch"
python generate_swatches.py "PLA Matte,BambuLab,Black" "PETG HF,BambuLab,White"
```

Each argument is `"Type,Producer,Color"`. Produces a 3MF with:

- Extruder 1 = text (black)
- Extruder 2+ = one per swatch body

Options: `-o filename.3mf`, `--keep` (preserve temp files), `--openscad /path`

The swatch design: 100x36x2mm card. Left half has 9 rectangle cutouts at varying depths (0.2-1.8mm) showing filament at different thicknesses. Right half has text area for producer/type/color labels in Arial Bold, auto-sized to fit.
